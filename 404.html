📖 Plain-English Glossary — what do all these terms mean?
▶ tap to open
📧 MX record
MX stands for "Mail Exchange". It is an entry in the internet's address book that says which servers handle email for this domain. If there are no MX records, the domain cannot receive email — so any email claiming to come from it is suspicious.
✅ SPF
SPF stands for "Sender Policy Framework". Think of it as a guest list for email. It is a record that says "only these servers are allowed to send email from our domain". Without it, anyone can fake an email pretending to be from that address.
🛡️ DMARC
DMARC stands for "Domain-based Message Authentication". It is a rule that tells receiving email servers what to do if someone tries to send a fake email using your domain — block it, send it to spam, or just report it. All major organisations have this.
🌐 DNS
DNS stands for "Domain Name System". It is the internet's address book — it translates website names (like google.com) into the actual server addresses (like 142.250.80.36) that computers use to connect. When we say "checking DNS" we mean looking up whether a domain actually exists and where it points.
🔒 SSL Certificate / HTTPS
An SSL certificate is what gives a website the padlock 🔒 in your browser. It means the connection is encrypted — like a sealed envelope. HTTPS (the "S" means secure) is the result of having one. While scam sites can also have HTTPS, not having it at all is a red flag.
🔐 Certificate Transparency
A public record of every SSL certificate ever issued. We check this to see how old a website's security history is. If a domain has certificates going back 3 years, it is almost certainly a real site. If it only has one certificate from last week, it is brand new — which matches how scam sites are created.
👀 Homoglyph / Look-alike letters
Some letters look almost identical. Scammers swap them to create fake addresses that look real. Common swaps: 1 (one) instead of l (lowercase L), 0 (zero) instead of o, rn instead of m. So "paypa1.com" looks like "paypal.com" at a glance.
📝 Typosquatting
Registering a domain name that is very similar to a real brand — like "amaz0n.com" or "goggle.com" — hoping people will mistype or not notice the difference. We measure how similar a domain is to real brands and flag it if it is suspiciously close.
🌍 Punycode / IDN
Some languages have letters that look identical to English ones but are technically different. Scammers register domains using, for example, Cyrillic letters that look exactly like Latin letters. To the eye it looks like "apple.com" but it is actually a completely different website. Punycode is how these domains are stored — they start with "xn--".
📊 AI Score / ML Model
The AI (machine learning) model looks at 16 things about the address — age, security history, name patterns and more — and combines them into a single probability score. It works like a checklist where each item is worth a different number of points. The final percentage is how likely the model thinks this is a scam.
🧠 How the AI learns
When you click "this was actually phishing" or "this was safe", the app slightly adjusts what it pays attention to — making it more accurate for similar cases next time. This is called "online learning". The adjustments are saved in your browser and stay between sessions. They are private to your device.
🔑 Risk vs Trust score
Every scan calculates two separate scores — Risk (bad signs add to this) and Trust (good signs add to this). The final verdict uses the difference: Risk minus Trust. This means a well-known domain can have a risk signal without being flagged, because its trust score is high enough to cancel it out.
